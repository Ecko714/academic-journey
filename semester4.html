  • And we're back! I am once again enrolled in classes after taking a semester off and I cannot wait to get back into the swing of things! Let's see what I learned this semester.
  • CSE 230 - Computer Org/Assem Lang Prog

    • Course Description

    "Register-level computer organization. Instruction set architecture. Assembly language. Processor organization and design. Memory organization. IO programming. Exception/interrupt handling"

    Alright. Time to get into the "nitty-gritty". This class went over high level assembly. Remember that item that was highlighted in red on a couple pages back with the word "Stack" on it? We are revisting that term in this class. That is because the stack data structure is an integral structure to any classical computer. Using the stack along with other technologies like interrupts, the CPU/ALU, and I/O, we learned how to write instructions for a CPU to use. Fascinating material with a good professor, one of the better courses that I've taken!

  • CSE 240 - Intro to Programming Languages

    • Course Description

    "This course introduces diverse paradigms of programming languages, including procedural (Language C), object-oriented (Language C++), functional (Language LISP), and logic (Language Prolog) paradigm"

    Another great course. This class gave me my first taste of C programming which was immensely helpful for other classes to come. Not only did it give me experience with programming in C, the course gave an overview of all different types of programming from procedural (C) to logic (prolog) programming. This was also one of the first courses that gave me a healthy challenge. My first programming course, CSE 110, wasn't too bad since I had prior programming experience and I already knew the basics. However, this course "pulled back the curtain" for me on programming and showed me more of what I was capable of. It was difficult but in a good way!

  • These are different programming "paradigms", which is a fancy way of saying different ideologies to programming.
    • Paradigm Description Languages
    Procedural/Imperative When you think of programming, this is most likely the paradigm you are thinking of. Programs written in this paradigm are meant to be "fully specified" and are executed in a "step-wise function". Key features of this paradigm include: modularized code (also known as functions, procedures, modules, or subroutines), local and global variables, data structures, and flow control statements like conditional and loop statements. C (~1972), Pascal (1970), Cobol (1959), Fortran (1954)
    Object-Oriented Programs written in this paradigm are taking advantage of a technology known as "encapsulation". Encapsulation allows for a programmer to bundle their code into "objects" that can be used throughout their program. This paradigm also comes with two other important features: Inheritance and Polymorphism. Inheritance is essentially a way to create new objects with new attributes from existing ones without rewriting code and Polymorphism allows for context-based use of objects.
    		Java (~1990), Smalltalk (1972), Simula (~1965)
    Function(al) Programmers who create projects using this paradigm put all of their code into functions. I've very little experience with this paradigm as it isn't very common in my studies. However, I don't mind because it can get very confusing for me personally (parentheses... so many parentheses...). Very mathematical in nature!
    		Haskell (1990), ML (1973), LISP (1959)
    Logic(al)/Declarative A program utilizing the logic paradigm will use a set of facts, rules, and questions about the relations between objects. The goal here is to create a create a database of relationships to then handle a series of queries, or questions, about the database. Again, not much experience here except for a few assignments.
    		Prolog (1972)
  • GLG 101 - Intro to Geology I (Physical)

    • Course Description

    "Basic principles of geology, geochemistry, and geophysics. Rocks, minerals, weathering, earthquakes, mountain building, volcanoes, water, and glaciers. Students must complete both GLG 101 and GLG 103 to receive an SQ (general studies)."

    Geology was more of a "let's get this over with" class for me, I'll be totally honest. I generally love learning, so it wasn't so bad, but it took up a lot of my time. The class had a couple of lectures every week and a lengthy textbook assignment coupled with a quiz. A lot of the work was more tedious than anything. We covered everything from minerals in the ground beneath us to understanding the surface of Mars.

  • HSE 101 - Human Systems Engineering

    • Course Description

    "This course is an introduction to the scientific study of how humans think, feel, and interact with technology, social systems, and each other"

    Oh my goodness, do I have some words for this class... I took this class because it sounds like the start to understanding the relationship between humans and technology, which I think sounds so cool to learn about. Nope, it was essentially a psychology 101 course. I give the course the benefit of the doubt since it is an introductory course and they couldn't jump into technical jargon without laying a foundation but, if I recall correctly, there were no lessons on application in tech. Regardless, we took the time to learn how the human brain works. Things like perception, consciousness, and personality were discussed. As was the case with geology, I enjoy learning so the class was still fascinating. I just wish there was a more technical application.
